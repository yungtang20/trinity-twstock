### 2. 4 個新查詢 API 端點
| 端點 | 功能 | 過濾條件 | 排序 |
|------|------|---------|------|
| GET /api/dashboard/recent-dividend | 最近一週發放股利 | dividend_events 近 7 天 | 日期 ASC, 現金股息 DESC |
| GET /api/dashboard/foreign-buy-2day | 投信連買二日 | trust_net >= 0 連續 2 天 | 成交量降冪 |
| GET /api/dashboard/break-ma200 | 突破 MA200 | 昨日收盤 < MA200, 今日收盤 > MA200 | 收盤價降冪 |
| GET /api/dashboard/limit-up-yesterday | 昨日涨停 | 漲幅 >= 9.8% | 爆量% 降冪 |
| GET /api/dashboard/db-health | 資料庫健康檢查 | 各表 COUNT + MAX(date) | - |

### 3. 資料來源選擇
- 預設使用本地 SQLite（效能最佳）
- 支援切換到 Supabase（透過 query parameter `?source=supabase`）
- FinMind 僅支援單檔查詢，篩選功能不適用

## 已知問題與待優化

1. **break-ma200 效能**: 對每檔股票個別查詢 MA200（N 次 SQL），約 6000+ 檔 × 2 次查詢 = 12000+ SQL。未來應改用 CTE 批次查詢。
2. **Supabase 查詢不完整**: 目前僅返回 stock_features 表的 limit 20 筆資料，非真正篩選邏輯。
3. **endpoints 名稱**: `foreign-buy-2day` 實際是投信連買，非外資。未來可考慮改名。
4. **limit-up-yesterday 爆量計算**: 使用 5 日均量，但 SQL 中 `ORDER BY date DESC LIMIT 5` 不保證取到 5 筆（可能少於 5 筆）。

## 技術細節

- Tailwind CSS v4: `@import "tailwindcss" source("./")` 確保 class 掃描
- SQLite WAL mode + busy_timeout 5000ms
- 所有端點統一回應格式: `{ success, data, error }`
- 股票篩選: `stock_id GLOB '[1-9][0-9][0-9][0-9]'` 排除 ETF/指數
