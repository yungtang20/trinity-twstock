## 1. 平台概述

TRINITY 台股分析平台是一個全方位的股票分析工具，提供即時行情、技術分析、籌碼分析、策略掃描和 AI 智慧分析功能。

### 核心功能
- **即時行情**：即時股票價格、漲跌幅、成交量
- **技術分析**：K 線圖、均線、支撐壓力、POC、VSBC
- **籌碼分析**：法人買賣超、千張大戶、集保人數
- **策略掃描**：支撐壓力、均線、籌碼、型態、預測
- **AI 智慧分析**：11 種華爾街分析模板

### 目標用戶
- 台股投資者
- 技術分析愛好者
- 量化交易者

### 4.2 API 呼叫流程

#### 4.2.1 單一報告生成流程

```
用戶點擊模板
    ↓
1. 驗證股票代號
    ↓
2. 收集 FinMind 數據
   ├─ 對每個數據集：
   │   ├─ 建立 API 請求參數
   │   │   ├─ dataset: 數據集名稱
   │   │   ├─ data_id: 股票代號（部分數據集不需要）
   │   │   ├─ start_date: 2015-01-01
   │   │   └─ end_date: 今日
   │   ├─ 發送 POST 請求到 /api/finmind-proxy
   │   ├─ 快取 API Token（避免重複認證）
   │   └─ 記錄結果（成功/失敗/無數據）
   ↓
3. 檢查數據是否足夠
   ├─ 如果無數據 → 返回「無數據」
   └─ 如果有數據 → 繼續
    ↓
4. LongCat 生成報告
   ├─ 建立提示詞
   │   ├─ 模板內容
   │   ├─ 股票代號
   │   └─ 可用數據（JSON 格式）
   ├─ 發送 POST 請求到 LongCat API
   │   ├─ model: LongCat-2.0-Preview
   │   ├─ max_tokens: 128,000
   │   └─ temperature: 0.7
   └─ 提取報告內容
    ↓
5. 顯示結果
```

#### 4.2.2 批量報告生成流程

```
用戶點擊「一鍵執行全部」
    ↓
1. 收集所有模板需要的數據集（去重）
   ├─ 建立數據集列表
   └─ 移除重複的數據集
    ↓
2. 一次性收集所有 FinMind 數據
   ├─ 對每個數據集：
   │   └─ 發送 API 請求
   └─ 記錄結果
    ↓
3. 逐個生成報告
   ├─ 對每個模板：
   │   ├─ 建立提示詞
   │   ├─ 發送 API 請求
   │   └─ 記錄結果
   └─ 更新進度
    ↓
4. 顯示所有結果
```

### 4.3 數據處理流程

#### 4.3.1 數據收集階段

```
開始
  ↓
對每個數據集：
  ├─ 建立 API 請求
  │   ├─ dataset: 數據集名稱
  │   ├─ data_id: 股票代號
  │   ├─ start_date: 2015-01-01
  │   └─ end_date: 今日
  │
  ├─ 發送請求
  │   └─ POST /api/finmind-proxy
  │       ├─ token: FinMind API Token
  │       └─ body: { dataset, data_id, start_date, end_date }
  │
  ├─ 處理回應
  │   ├─ 成功 → 儲存數據
  │   ├─ 失敗 → 記錄錯誤
  │   └─ 無數據 → 記錄無數據
  │
  └─ 記錄日誌
      ├─ [✓] dataset: N 筆
      ├─ [○] dataset: 無數據
      └─ [✗] dataset: 調閱失敗
```

#### 4.3.2 數據格式化階段

```
原始數據
  ↓
轉換為 JSON 格式
  ├─ 每個數據集：
  │   ├─ dataset: 數據集名稱
  │   └─ data: 數據陣列
  │       └─ 每筆數據：{ date, open, high, low, close, volume, ... }
  │
  └─ 合併所有數據集
      └─ allData: [{ dataset, data }, ...]
```

#### 4.3.3 提示詞生成階段

```
模板內容
  ↓
加入股票代號
  ├─ 股票代號：2330
  └─ 股票名稱：台積電
  ↓
加入可用數據
  ├─ 數據集 1：TaiwanStockPrice（1,095 筆）
  ├─ 數據集 2：TaiwanStockFinancialStatements（50 筆）
  └─ ...
  ↓
加入分析要求
  ├─ 只接受數據分析
  ├─ 不編造內容
  ├─ 若數據不足，回答「無數據」
  └─ 禁止使用模擬數據
  ↓
最終提示詞
```

### 4.4 LongCat API 呼叫流程

#### 4.4.1 請求格式

```json
{
  "model": "LongCat-2.0-Preview",
  "messages": [
    {
      "role": "system",
      "content": "你是一位專業的台股分析師。根據提供的數據，生成詳細的分析報告。只基於數據回答，不要編造。"
    },
    {
      "role": "user",
      "content": "分析需求：\n${template.prompt}\n\n股票代號：${stockId}\n\n可用數據：\n${JSON.stringify(data, null, 2)}\n\n只接受數據分析，不編造內容。若數據不足，回答「無數據」。禁止使用模擬數據，一定要是從 FinMind 取得的數據。"
    }
  ],
  "temperature": 0.7,
  "max_tokens": 128000
}
```

#### 4.4.2 回應格式

```json
{
  "choices": [
    {
      "message": {
        "content": "分析報告內容..."
      }
    }
  ]
}
```

#### 4.4.3 回應處理

```
接收回應
  ↓
提取 choices[0].message.content
  ↓
如果 content 為空 → 返回「無數據」
  ↓
否則 → 返回分析報告
```

### 4.5 錯誤處理流程

#### 4.5.1 數據收集錯誤

```
API 請求失敗
  ├─ 網路錯誤 → 記錄錯誤，繼續下一個數據集
  ├─ API 限制 → 記錄錯誤，繼續下一個數據集
  └─ 數據格式錯誤 → 記錄錯誤，繼續下一個數據集
```

#### 4.5.2 LongCat API 錯誤

```
API 請求失敗
  ├─ 網路錯誤 → 顯示錯誤訊息
  ├─ API 限制 → 顯示錯誤訊息
  └─ 回應格式錯誤 → 顯示錯誤訊息
```

#### 4.5.3 數據不足

```
如果收集到的數據為空
  ├─ 記錄日誌：[完成] 無數據
  ├─ 顯示結果：「無數據」
  └─ 不調用 LongCat API
```
