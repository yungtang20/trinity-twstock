## 測試標的類型

每種策略測試時，至少涵蓋以下 7 種標的：

| 類型 | 範例 | 測試目的 |
|------|------|---------|
| 大盤權值股 | 2330（台積電） | 流動性高、資料完整 |
| ETF | 0050（元大台灣50） | 無除權息干擾、趨勢明確 |
| 上櫃股 | 2317（廣達） | 確認 OTC 資料正常 |
| 金融股 | 2881（富邦金） | 股價低、波動小 |
| 高波動股 | 3711（創意） | 測試極端值處理 |
| 除權息股 | 2327（鴻海） | 測試 dividend_events 寫入 |
| 停牌/處置股 | 依當日查詢 | 測試空值 / 異常處理 |

---

## 測試資料要求

### 基本資料

每支測試標的必須具備：

- 至少 100 筆歷史 K 線資料
- 至少 5 筆除權息事件
- 至少 10 筆三大法人資料
- 至少 3 筆集保資料

### 資料完整性檢查

```sql
-- 檢查測試標的資料完整性
SELECT
    stock_id,
    COUNT(CASE WHEN volume > 0 THEN 1 END) as active_days,
    COUNT(CASE WHEN volume = 0 THEN 1 END) as zero_volume_days,
    MIN(date) as first_date,
    MAX(date) as last_date
FROM stock_history
WHERE stock_id IN ('2330', '0050', '2317', '2881', '3711', '2327')
GROUP BY stock_id;
```

### 預期結果

| 測試 | 預期行為 |
|------|---------|
| 2330 | 正常輸出，無 error |
| 0050 | 正常輸出，無 error |
| 2317 | 正常輸出，無 error |
| 2881 | 正常輸出，無 error |
| 3711 | 正常輸出，無 error |

## 新增測試標的

1. 新增測試標的時，必須更新此文件的表格
2. 至少涵蓋一種新類型（例如：興櫃、優先股、新上市）
3. 確認該標的在 DB 中有足夠資料
